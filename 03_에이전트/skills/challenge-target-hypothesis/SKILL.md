# 타깃 가설 반증

## 절차

1. 최초 가설과 채택 이유를 한 문장으로 고정한다.
2. 승인 약물이 있다면 승인 적응증과 현재 질병이 같은지 먼저 확인한다.
3. 현재 적응증의 실패·중단·유효성 미달 임상을 우선 검색한다.
4. 유전 근거, 임상 근거, tractability, assay density, 안전성 신호를 서로 독립적으로 평가한다.
5. 모순이 발견되면 어떤 최초 전제를 무효화하는지 연결한다.
6. 반증 이후 가설을 유지, 하향, 보류, 기각 중 하나로 권고한다.

## 필수 사례

IBD/TYK2/deucravacitinib에서는 psoriasis 승인을 IBD 검증으로 사용하지 않는다. 현재 고정 근거 패키지에서는 실패한 IBD 임상 때문에 `REJECT` 또는 검토가 필요한 `HOLD`를 권고한다.

## 출력

- 최초 주장과 수정 주장
- contradiction ledger
- 영향받은 점수 요소
- 남은 불확실성
- 권고 상태와 다음 검증 행동

근거가 없다는 사실과 반증 근거가 있다는 사실을 구분한다. 내부 사고과정 대신 검증 가능한 근거와 규칙만 노출한다.

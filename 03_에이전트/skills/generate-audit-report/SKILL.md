# 감사 보고서 생성

## 구성

1. 질병 질의와 정규화 결과
2. 한 문장 가설과 최종 `ADVANCE/HOLD/REJECT`
3. 지지 근거와 반증 근거 표
4. 적용된 결정 규칙과 점수 변화
5. 분자 단계 진입 여부와 실행·미실행 사유
6. 후보별 measured/QSAR/proxy 구분과 검증 결과
7. failure ledger, API fallback, 누락된 검증
8. source ID, URL, 관측일, snapshot hash
9. 인간 검토 상태와 다음 실험·조사 제안

## 표현 규칙

- 결론보다 근거와 실패를 먼저 검증한다.
- 다른 적응증의 승인, 예측값, 유사도 proxy를 측정 효능으로 표현하지 않는다.
- 내부 사고과정을 노출하지 않고 외부 검증 가능한 근거·규칙·결과만 설명한다.
- synthetic UI fixture와 실제 연구 결과를 명확히 분리한다.
- 실행하지 않은 ADMET·QSAR·역합성을 실행한 것처럼 쓰지 않는다.

보고서는 간결한 요약과 감사 가능한 부록을 함께 제공한다.
